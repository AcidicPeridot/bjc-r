Redundancy

An important feature of networks is redundancy: multiple paths to get to the same place. If one path is no longer available for some reason, a message can travel a different path instead.

Consider roads and highways: we might always go to a destination (like a school) using a certain path. However, if one day, one of the roads is under construction, we can usually find an alternative way to get to the destination. The alternative route might be slower or more dangerous, but at least we eventually get to the destination! That's the power of redundancy.

Break the Network

More redundancy in a network means more protection in case things go wrong in the network. This network has some redundancy, but not a lot, so there's a lot that can go wrong.

Now you're going to cause some havoc!

There are several blocks for taking down parts of the network:

The cut cable blocks simulates something that actually happens a lot, cutting the cables connecting two routers. Many cables are either buried underground or lying on the seafloor, so they can easily be damaged by construction companies or ships that don't realize they're there. In 2019, a cut cable resulted in the island nation of Tonga losing it's entire Internet connection!

The other blocks simulate the taking down of individual routers, not the cables between them. Natural disasters are a common cause of router loss, since routers can be destroyed or lose power. We have yet to see a router be destroyed by a T-Rex, but hey, you never know!
