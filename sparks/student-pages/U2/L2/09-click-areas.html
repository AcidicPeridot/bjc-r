Creating Click Areas

  • Pamela's project
In this activity, you will decide which parts of the stage background will respond to user clicks and what will happen when a user clicks them.

There are many reasons to use clicks on specific areas of the stage background to run code. Clicking on part of a virtual poster might show information. Clicking part of a game map could move the player.

Locations on the Snap! stage are described using x and y coordinates. The x coordinate tells you the horizontal position. The y coordinate tells you the vertical position. To define a click area, you will need two x coordinates (the right and left sides) and two y coordinates (the top and bottom).

Broadcasting a Click Message

In this activity, you will use messages to tell other parts of your program what to do when the user clicks your click area.

Since your project will react to clicks on the stage, you'll need to create code for the stage. You can find the stage scripts by clicking the "Stage" icon in the lower left corner of the Snap! window. (You can return to your sprite code by clicking a sprite icon.)

The code for the stage controls only the stage (such as its background). The code for each sprite controls only that sprite (such as its costume or movement). If you want clicking the stage to make a sprite do something, then you need to make the stage talk to the sprite.

In Snap!, messages let one part of your program talk to another part of your program. The broadcast block sends a message to every part of your project. You can set the message to anything that will help you remember what it means. Here's an example:
